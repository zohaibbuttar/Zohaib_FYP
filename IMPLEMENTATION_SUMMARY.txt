═══════════════════════════════════════════════════════════════════════════════
  HOW IT WORKS
═══════════════════════════════════════════════════════════════════════════════

Customer Sign Up & Login Flow:
   1. User goes to /auth/sign-up
   2. Fills in email, password, name, phone
   3. Submits form (role automatically "customer")
   4. SQL trigger creates profile with role='customer'
   5. Email redirect confirmation (or auto-verified)
   6. User redirected to /auth/sign-up-success
   7. User can click link to login at /auth/login
   8. After login, redirected to /dashboard
   9. Cannot access /admin (middleware redirects to /dashboard)

Owner Login Flow:
   1. Owner (admin) goes to /auth/owner-login
   2. Enters email and password
   3. System checks role == 'admin'
   4. If not admin: Shows error "Access denied. Owner account required."
   5. If admin: Redirects to /admin
   6. Admin can see admin dashboard

Manual Admin Assignment:
   1. Owner signs up as customer first
   2. Admin runs SQL: UPDATE profiles SET role='admin' WHERE email='...'
   3. Owner logs in at /auth/owner-login
   4. Now has access to /admin

Route Protection:
   /admin without login     → redirects to /auth/login
   /admin as customer       → redirects to /dashboard
   /admin as admin          → allowed
   /dashboard without login → redirects to /auth/login
   /dashboard as customer   → allowed
   /dashboard as admin      → allowed
